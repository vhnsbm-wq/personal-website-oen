## 操作步骤

### 步骤 1：创建 public 目录

在项目根目录创建 `public` 文件夹（如果还没有）：

```
d:\kf\personal website\
  ├── public\          ← 创建这个文件夹
  ├── src\
  ├── package.json
  └── ...
```

### 步骤 2：保存截图

1. **保存你提供的截图**
   - 将截图保存为 `markdown-editor.png` 或 `markdown-editor.jpg`
   - 放到 `public` 目录下

2. **文件路径应该是：**
   ```
   d:\kf\personal website\public\markdown-editor.png
   ```

### 步骤 3：更新项目数据

打开 `src/data/projects.ts`，确认图片路径：

```typescript
{
```

## 详细操作（Windows）

### 方法 1：通过文件资源管理器

1. **打开项目文件夹**
   - 在文件资源管理器中打开：`d:\kf\personal website`

2. **创建 public 文件夹**
   - 右键 → 新建 → 文件夹
   - 命名为 `public`

3. **保存截图**
   - 将你的截图文件复制到 `public` 文件夹
   - 重命名为 `markdown-editor.png`

### 方法 2：通过 VS Code

1. **在 VS Code 中**
   - 在左侧文件树中，右键项目根目录
   - 选择 "New Folder"
   - 输入 `public`

2. **添加图片**
   - 将截图文件拖拽到 `public` 文件夹
   - 或右键 `public` → "Reveal in File Explorer"
   - 然后复制图片到该文件夹

## 图片命名建议

推荐使用以下命名：
- `markdown-editor.png`（推荐）
- `markdown-editor.jpg`
- `project1.png`

**注意：**
- 文件名使用小写字母
- 使用连字符 `-` 而不是空格
- 使用常见的图片格式（png, jpg, webp）

## 更新 projects.ts

如果你的图片名称不是 `markdown-editor.png`，需要更新 `src/data/projects.ts`：

```typescript
image: '/你的图片名称.png',
```

例如：
- `/markdown-editor.png`
- `/markdown-editor.jpg`
- `/project-screenshot.png`

## 测试图片是否正确

### 1. 本地测试

```bash
# 启动开发服务器
npm run dev
```

访问 http://localhost:5173/，滚动到"项目展示"部分，检查图片是否显示。

### 2. 检查图片路径

如果图片不显示：
1. 确认图片在 `public` 目录
2. 确认文件名拼写正确
3. 确认 `projects.ts` 中的路径正确
4. 刷新浏览器（Ctrl + F5）

## 图片优化建议

### 1. 图片尺寸

推荐尺寸：
- 宽度：1200-1600px
- 高度：600-900px
- 比例：16:9 或 4:3

### 2. 图片大小

- 尽量控制在 500KB 以内
- 使用在线工具压缩：[tinypng.com](https://tinypng.com/)

### 3. 图片格式

- **PNG**：适合截图，支持透明背景
- **JPG**：适合照片，文件更小
- **WebP**：现代格式，体积最小（推荐）

## 如果图片太大

### 使用在线工具压缩

1. 访问 [tinypng.com](https://tinypng.com/)
2. 上传你的截图
3. 下载压缩后的图片
4. 保存到 `public` 目录

### 使用截图工具

如果需要重新截图：
1. 访问你的 Markdown 编辑器
2. 按 `F12` 打开开发者工具
3. 按 `Ctrl + Shift + P`
4. 输入 "screenshot"
5. 选择 "Capture full size screenshot"

## 常见问题

### Q1: 图片不显示

**检查：**
1. 图片是否在 `public` 目录
2. 文件名是否正确（区分大小写）
3. 路径是否以 `/` 开头
4. 浏览器是否缓存了旧页面（强制刷新：Ctrl + F5）

### Q2: 图片显示但很模糊

**解决：**
- 使用更高分辨率的截图
- 推荐至少 1200px 宽度

### Q3: 图片加载很慢

**解决：**
- 压缩图片大小
- 使用 WebP 格式
